Bucket Details

This page allows you to view details about the bucket and edit the sharing properties of a bucket.

Bucket summary

The Bucket section provides a summary of the bucket, including the number of objects it contains.

Actions menu

Clicking the Actions menu displays the options to view the contents of your bucket and delete the bucket.

View bucket contents

The View contents option allows you to see the objects (files and folders) in each bucket. You can also upload new files, create new folders, or delete existing objects.

Delete a bucket

The Delete option allows you to delete the bucket if it is empty.

Edit the sharing settings

The sharing settings dictate whether your account or another account can acccess your bucket and its contents.
  1. To edit the sharing settings, select from the following:
    1. Select Public to allow everyone access your bucket and its contents.
      Important: Making a bucket public means anyone who has the URL can access that bucket and its contents, even if they are not authenticated users.
    2. Select Private to allow only those specified to access your bucket and its contents.
      If the selection changes the current settings, an informational message displays stating that the sharing settings will propagate to all objects in this bucket.
    3. Click Yes, Let's Do This to acknowledge the message and continue.
      Note: Because this message is informational only, clicking the Yes, Let's Do This button or clicking the X from the corner of the message window, or clicking outside the message window, no action is applied to the objects in the bucket.
      The Share with others section displays only if Private was chosen as the Sharing option for this bucket.
  2. Specify access control parameters by performing the following:
    1. Select Use canned ACL to use a pre-defined Access Control List established by your organization.
      Newly-created buckets have a default sharing setting of Private Full Control.
      Note: For more information about ACL, go to Access Control List (ACL) Overview.
    2. To change to another pre-defined ACL, select a pre-defined ACL from the Use canned ACL drop-down list box.
    3. Select Manually define sharing to grant only specific accounts certain levels of access to your bucket.
      Note: In order to manually define sharing with other accounts, specify that account with an account ID or an email address associated with a user in the account. To obtain an account ID, coordinate with the owners or administrators of those accounts. Otherwise, specifying an email address of a user in the account will effectively grant access to everyone in that account.
      Note: You can remove an existing account and permission pair by clicking on the x next to the pair listed under the Manually define sharing option.
    4. Enter the 12-digit account ID or email address of a user in the account in the Account field.
    5. Select the appropriate level of access for the account by selecting it from the Permissions drop-down list box.
    6. Click the Add Account or Add another account button (if one or more accounts were already present) to add it to the list of accounts with which your bucket is shared.
  3. When done, click the Save Changes button to save your work, or click Cancel to abandon your changes.
