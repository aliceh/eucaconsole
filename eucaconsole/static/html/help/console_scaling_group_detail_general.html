Scaling Group Detail - General

An Auto Scaling group defines the parameters for the Eucalyptus instances that are used for scaling, as well as the minimum, maximum, and desired number of instances to use for Auto Scaling your application. This page allows you to view and edit a scaling group. For more information on Auto Scaling, please see Using Auto Scaling in the Eucalyptus User Guide.

Capacity

This tab is where you specify the minimum, maximum, and desired number of instances your scaling group will keep running.

  1. Select the maximum number of instances you want running in your auto scaling group using the Maximum control.
  2. Specify the minimum number of instances you want running using the Minimum control.
  3. Specify the desired number of instances you want running using the Desired control.
  4. A termination policy defines how instances that are no longer needed in the scaling group are terminated. Select one or more termination policies from the Termination policies drop-down list box. The termination policies will be executed in the order you select them.

Details

You can change the scaling group's launch configuration, availability zones, and cooldown period here.

  1. Select a launch configuration from the Launch Configuration drop-down list box. A launch configuration defines the properties of the instances that are launched as part of your auto scaling group.
  2. You can update the availability zones for your scaling group using the Availability zones drop-down list box.
    You can remove a selected availability zone by clicking on the x next to the availability zone's name in the Availability Zones field.
  3. The cooldown period is the number of seconds after a scaling activity completes before any new scaling activity can start. Use the Default cooldown period (seconds) control to select the cooldown period for your auto scaling group.

Health Check

The auto scaling health check uses EC2 instance status checks to determine the health state of each instance in your auto scaling group. Use this section to change the type of health check used for your scaling group.

  1. Select the type of health check for your auto scaling group from the Type drop-down list box.
  2. Type the amount of grace period, in seconds, into the Grace period text box. This is the amount of time after a new instance is launched in your auto scaling group before health checks start for the instance.

Tags

A tag is a key/value pair containing data that you can attach to resources in your cloud. This section allows you to view, add or delete tags for for your scaling group and for instances that run in your scaling group.

  1. If you want to apply the new tag to instances running in the scaling group, select the Propagate check box.
  2. Type the key name for your tag into the Key text box.
  3. Type the value for your tag into the Value text box.
  4. If you wish to add additional tags, click the Add Tag button.
  5. If you wish to delete a tag that you've already entered, click the x on the tag icon.

Saving Your Changes

Once you're satisfied with the edits you've made to your scaling group, click the Save changes button, or click the x button in the upper right corner to cancel the updates.
